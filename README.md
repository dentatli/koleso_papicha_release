# Колесо для аукциона донатов

Локальное приложение для проведения донат-аукциона и прокрутки колеса на выбывание.

Проект предназначен для стримов: можно добавлять лоты, принимать донаты из DonatePay и DonationAlerts, назначать донаты на лоты и крутить колесо в отдельной вкладке для OBS/захвата экрана.

## Возможности

* Ведение списка лотов для аукциона
* Добавление, удаление и редактирование лотов
* Учет суммы по каждому лоту
* Автоматический расчет шансов
* Колесо на выбывание
* Отдельный режим отображения колеса
* Очередь донатов:

  * новые донаты
  * обработанные донаты
  * игнорированные донаты
* Подключение DonatePay
* Подключение DonationAlerts
* Тестовые донаты для проверки без реальной оплаты
* Таймер
* Настройки масштаба интерфейса
* Локальное сохранение состояния в браузере
* Локальный PowerShell-сервер на `127.0.0.1:5500`

## Быстрый запуск

1. Скачайте архив проекта.
2. Распакуйте его в отдельную папку.
3. Запустите файл:

```bat
start_koleso.bat
```

4. Откроется локальная страница приложения:

```text
http://127.0.0.1:5500/koleso_papich.html?view=admin
```

## Режимы работы

### Админка

Админка используется для управления аукционом:

* добавление лотов;
* редактирование сумм;
* подключение донат-сервисов;
* просмотр очереди донатов;
* назначение донатов на лоты;
* управление таймером;
* запуск колеса.

Админку нельзя показывать на стриме, потому что в ней могут быть настройки интеграций и токены.

### Колесо

Отдельная вкладка колеса предназначена для OBS или другого захвата экрана.

Открывается кнопкой:

```text
Открыть колесо в отдельной вкладке
```

Обычно используется адрес вида:

```text
http://127.0.0.1:5500/koleso_papich.html?view=wheel
```

## Подключение DonatePay

1. Откройте вкладку `Настройки`.
2. В блоке `Донат-сервисы` выберите `DonatePay`.
3. Нажмите `Подключить`.
4. Выберите регион:

   * Россия — `donatepay.ru`
   * Европа — `donatepay.eu`
5. Вставьте API Key из DonatePay.
6. Нажмите `Подключить`.

После подключения новые донаты будут попадать в очередь `Донаты`.

## Подключение DonationAlerts

1. Откройте вкладку `Настройки`.
2. В блоке `Донат-сервисы` выберите `DonationAlerts`.
3. Нажмите `Подключить`.
4. Выберите быстрый вход или подключение через свое OAuth-приложение.
5. Разрешите доступ на сайте DonationAlerts.
6. После возврата в приложение дождитесь статуса подключения.

Приложение принимает только новые донаты после запуска/подключения, чтобы не подтягивать старую историю как новые события.

## Как работать с донатами

Новые донаты появляются во вкладке:

```text
Аукцион → Донаты → В ожидании
```

Для каждого доната можно:

* выбрать лот;
* назначить сумму на выбранный лот;
* проигнорировать донат.

После обработки донат переносится во вкладку:

```text
Добавлено
```

## Тестовый донат

Для проверки работы без реальной оплаты:

1. Откройте `Настройки`.
2. Найдите блок `Тестовый донат`.
3. Укажите имя, сумму и сообщение.
4. Нажмите `Добавить в ожидание`.
5. Перейдите во вкладку `Аукцион`.
6. Назначьте тестовый донат на нужный лот.

## Сохранение данных

Приложение сохраняет состояние локально в браузере:

* список лотов;
* суммы;
* состояние колеса;
* донаты;
* настройки отображения;
* настройки интеграций;
* состояние таймера.

Данные хранятся в `localStorage`.

Если очистить данные сайта в браузере, состояние приложения будет сброшено.

## Сброс данных

Во вкладке `Настройки` есть кнопка:

```text
Сбросить все данные
```

Она удаляет:

* лоты;
* донаты;
* подключения;
* настройки;
* состояние таймера.

Используйте эту кнопку осторожно. Действие нельзя отменить.

## Безопасность

Проект является локальным инструментом, а не полноценным публичным backend-сервисом.

Важные правила:

* Не показывайте вкладку `Настройки` на стриме.
* Не показывайте API Key DonatePay.
* Не показывайте OAuth-данные DonationAlerts.
* Не публикуйте свою локальную копию проекта с уже заполненными токенами.
* Не запускайте проект из неизвестного архива, если не доверяете источнику.
* Не держите локальный сервер включенным без необходимости.

Локальный сервер слушает адрес:

```text
127.0.0.1:5500
```

Это означает, что приложение рассчитано на работу только на вашем компьютере.

## Структура проекта

Основные файлы:

```text
start_koleso.bat       — запуск приложения
local_server.ps1       — локальный PowerShell-сервер
koleso_papich.html     — основной интерфейс приложения
README.md              — описание проекта
```

В release-сборке файлы приложения могут находиться в папке `assets`.

Пример структуры release-сборки:

```text
start_koleso.bat
assets/
  koleso_papich.html
  local_server.ps1
  tray.ico
  tray-error.ico
```

## Что делает `start_koleso.bat`

Файл запускает локальный PowerShell-сервер и открывает приложение в браузере.

Пример запуска:

```bat
powershell.exe -NoLogo -NoProfile -STA -ExecutionPolicy Bypass -WindowStyle Hidden -File "%~dp0assets\local_server.ps1" -Root "%~dp0assets"
```

`ExecutionPolicy Bypass` используется для запуска локального скрипта без ручной настройки политики PowerShell.

## Что делает `local_server.ps1`

Локальный сервер:

* запускает HTTP-сервер на `127.0.0.1:5500`;
* раздает файлы интерфейса;
* открывает приложение в браузере;
* проксирует запросы к DonatePay;
* проксирует запросы к DonationAlerts;
* собирает новые донаты;
* ведет локальный лог;
* проверяет наличие новой версии проекта;
* защищает раздачу файлов от выхода за папку проекта.

## Лог приложения

Сервер может создавать файл:

```text
server.log
```

В нем сохраняются технические сообщения о запуске, ошибках подключения и работе collector.

Секретные значения в логах должны маскироваться.

## Возможные проблемы

### Приложение не открывается

Проверьте, что порт `5500` свободен.

Если порт занят, закройте другую копию приложения или процесс, который использует этот порт.

### Браузер не открылся автоматически

Откройте адрес вручную:

```text
http://127.0.0.1:5500/koleso_papich.html?view=admin
```

### DonationAlerts не подключается

Проверьте:

* запущен ли `start_koleso.bat`;
* открыт ли сайт именно через `127.0.0.1:5500`;
* правильно ли указан redirect URL;
* есть ли доступ к сайту DonationAlerts;
* не истек ли токен авторизации.

### DonatePay не подключается

Проверьте:

* правильно ли выбран регион;
* корректный ли API Key;
* доступен ли DonatePay;
* не превышен ли лимит запросов;
* не блокируется ли WebSocket соединение.

### Донаты не появляются

Проверьте:

* включена ли интеграция;
* есть ли статус подключения;
* не были ли донаты уже обработаны;
* появились ли донаты после запуска приложения;
* нет ли ошибок в `server.log`.

## Ограничения

* Это локальный MVP, а не публичный сервер.
* Токены сохраняются локально в браузере.
* Приложение не предназначено для размещения на публичном хостинге без доработки backend-части.
* Для надежной публичной версии нужен отдельный backend, авторизация, безопасное хранение токенов и нормальная серверная база данных.

## Рекомендации для стрима

* В OBS захватывайте только вкладку `Колесо`.
* Админку держите на отдельном мониторе или скрытой вкладке.
* Перед началом стрима проверьте тестовый донат.
* Не открывайте вкладку `Настройки` на трансляции.
* Не показывайте `server.log`, если там могут быть технические данные.
