# Экспериментальный тестер AI-распознавания

Изолированный инструмент для последовательной проверки сообщений донатов через OpenRouter. Он не подключается к DonationAlerts/DonatePay, не читает аукцион, не создаёт лоты и не использует production AI jobs или каталожные кэши.

## Запуск

Запустите `start_ai_tester.bat`. Откроется `http://127.0.0.1:5501/`. Основное приложение на порту 5500 может работать одновременно.

Тестер использует существующий OpenRouter API key и HTTP(S)-прокси из `%LOCALAPPDATA%\PapichWheel\secrets.json`. Секреты расшифровываются только серверным процессом и не передаются странице. Если ключ ещё не настроен, настройте его в основном приложении.

Сохранённый экспериментальный промпт и журнал тестера находятся отдельно:

```text
%LOCALAPPDATA%\PapichWheel\ai-tester\
```

Очередь и результаты сохраняются в отдельном ключе localStorage origin `127.0.0.1:5501`. После перезагрузки выполнение не начинается автоматически.

## Форматы ввода

- JSON-массив или `{ "donations": [] }`.
- Поля `name|username|donorName`, `amount|sum`, `message|comment|text`, `id|externalId`.
- Обычный текст: одна непустая строка — один донат.
- Если присутствует строка `---`, блоки между разделителями считаются многострочными донатами.

Один элемент очереди создаёт один основной OpenRouter-запрос. Запросы никогда не выполняются параллельно. Rate-limit может вызвать ограниченный последовательный retry с учётом `Retry-After`.

## Безопасность

- Сервер слушает только `127.0.0.1`.
- API защищён случайным токеном текущего процесса и same-origin проверкой.
- Экспорт формируется по allowlist и не содержит API key, proxy URL, local token или путей.
- Сырой AI content ограничен 64 KiB.
- Пользовательские строки выводятся через `textContent`.

## Тесты

```text
node tests/ai_tester_logic_tests.js
powershell.exe -NoProfile -ExecutionPolicy Bypass -File tests/ai_tester_server_tests.ps1
```

Автоматические тесты используют mocks и не вызывают OpenRouter.
